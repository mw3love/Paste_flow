- 질문이 있으면 적극적으로 할것
- 생성된 md문서를 프로젝트 Fix폴더 안에 생성하고, 문서의 제목은 'yymmdd_내용' 형식으로 한다.
- [중요] 항상 대화 컨텍스트를 고려하여 단계별로 작업량을 나눈다.
- 각 단계마다 todo list를 만들어서 중간에 중단되더라도 이어서 작업할 수 있도록 작성한다.
- 각 list별로 기능 구현한뒤 기능이 잘 작동하는지 자체 테스트
- 완료된 항목들은 문서에 바로 업데이트한다.
- 핵심내용이나 규칙같은 중요 변경은 사용자에게 물어본뒤 CLAUDE.md나 PRD.md같은 관련문서에 업데이트 할것

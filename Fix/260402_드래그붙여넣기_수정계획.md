# 드래그 붙여넣기 미동작 수정계획

## 문제 현상
- 패널에서 항목을 드래그해 외부 앱에 놓으면 붙여넣기가 되지 않음
- Win10 메모장(클래식 Win32)에서는 동작 확인됨
- **Win11 메모장**에서 미동작 (항상 재현)
- **노션(Electron)**에서 미동작 (항상 재현)
- 기타 Electron/Chromium 기반 앱 동일 예상

## 원인 분석

### 1순위: 창 탐색 1레벨만 수행 — Win11 메모장 미동작
- 파일: `pasteflow/main.py` L281-284
- `ChildWindowFromPoint(hwnd, client_pt)` 는 **직계 자식 1레벨만** 반환
- Win11 메모장은 WinUI3/XAML Islands 기반으로 창 계층이 깊음:
  ```
  Notepad (top HWND)
    └─ DesktopChildSiteBridge (XAML host)
         └─ Windows.UI.Composition...
              └─ RichEditD2DPT  ← WM_PASTE가 실제로 필요한 컨트롤
  ```
- 현재 코드는 `DesktopChildSiteBridge` 수준에서 멈춰 `WM_PASTE`가 전달되지 않음

### 2순위: WM_PASTE 메시지 자체가 무효 — Electron/Chromium 미동작
- 파일: `pasteflow/main.py` L288
- `win32gui.SendMessage(target, win32con.WM_PASTE, 0, 0)`
- Electron 앱은 텍스트 렌더링을 Chromium GPU 프로세스가 담당
- `WM_PASTE`는 Win32 EDIT/RichEdit 컨트롤 전용 메시지 — Chromium 렌더러는 처리하지 않음
- 아무리 정확한 HWND를 찾아도 `WM_PASTE` 방식 자체가 동작하지 않음
- **필요한 접근**: 클립보드 설정 후 타겟 창을 활성화 → `SendInput(Ctrl+V)`
  - CLAUDE.md 금지 사항: "백그라운드 스레드에서 포그라운드 잠금에 막혀 실패"
  - 드래그 핸들러는 **Qt 메인 스레드**에서 실행 → `AttachThreadInput` 트릭으로 포그라운드 잠금 우회 가능

### Phase 1 — 재귀 창 탐색으로 Win11 메모장 수정

**변경 위치**: `pasteflow/main.py` `_on_drag_to_app()` 내 창 탐색 부분

**구현**:
- `ChildWindowFromPoint` 1회 호출 → 재귀 탐색 함수 `_find_deepest_child(hwnd, screen_pt)` 으로 교체
- 재귀 로직: `ChildWindowFromPoint`로 자식 찾음 → 자식이 있으면 다시 그 자식부터 탐색 → 더 이상 내려가지 않을 때까지 반복
- **무한루프 방지** (Critical 이슈): `visited` set + `MAX_DEPTH = 20` 적용

```python
def _find_deepest_child(hwnd, screen_pt, visited=None, depth=0):
    if visited is None:
        visited = set()
    if depth > 20 or hwnd in visited:
        return hwnd
    visited.add(hwnd)
    try:
        client_pt = win32gui.ScreenToClient(hwnd, screen_pt)
        child = win32gui.ChildWindowFromPoint(hwnd, client_pt)
        if child and child != hwnd:
            return _find_deepest_child(child, screen_pt, visited, depth + 1)
    except Exception:
        pass
    return hwnd
```

### Phase 2 — 창 클래스 기반 방식 분기로 Electron/Notion 수정

**변경 위치**: `pasteflow/main.py` `_on_drag_to_app()` 붙여넣기 실행 부분

**구현 전략**: 타겟 창의 Win32 클래스명을 읽어 두 방식 중 하나 선택

| 타겟 창 클래스 | 방식 | 대상 앱 |
|---|---|---|
| `EDIT`, `RichEdit*`, `RichEditD2DPT`, `Scintilla` 등 | `WM_PASTE` | 클래식 메모장, Win11 메모장, VSCode 에디터 내부 등 |
| `Chrome_RenderWidgetHostHWND`, `Chrome_WidgetWin_*`, `CefBrowserWindow`, `CEF*` 등 | AttachThreadInput + SetForegroundWindow + delayed SendInput | 노션, Slack, VS Code(Electron 셸), Chrome 등 |
| 기타 알 수 없는 클래스 | WM_PASTE 시도 후 fallback으로 AttachThreadInput 방식 | 범용 fallback |

**AttachThreadInput 방식 구현**:

```python
def _activate_and_send_ctrl_v(hwnd):
    import win32process, win32con
    import ctypes

    fg_hwnd = win32gui.GetForegroundWindow()
    current_tid = ctypes.windll.kernel32.GetCurrentThreadId()
    fg_tid = win32process.GetWindowThreadProcessId(fg_hwnd)[0]
    target_tid = win32process.GetWindowThreadProcessId(hwnd)[0]

    # 포그라운드 잠금 우회: 현재 스레드를 fg 스레드에 attach
    win32process.AttachThreadInput(current_tid, fg_tid, True)
    win32gui.SetForegroundWindow(hwnd)
    win32gui.BringWindowToTop(hwnd)
    win32process.AttachThreadInput(current_tid, fg_tid, False)

    # 창 활성화 완료 후 Ctrl+V 전송 (50ms 지연)
    QTimer.singleShot(50, _send_ctrl_v_input)

def _send_ctrl_v_input():
```

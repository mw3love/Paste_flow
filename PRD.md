# PasteFlow — 순차 붙여넣기 클립보드 매니저 PRD

> **Product Requirements Document**
> 버전: 1.0 | 작성일: 2026-03-22 | 플랫폼: Windows 10/11

---

## 1. 개요 (Overview)

### 1.1 목적
Windows에서 복사한 항목들을 **복사한 순서대로 자동으로 순차 붙여넣기**하는 클립보드 매니저. 별도의 모드 전환 없이, 복사한 순서 그대로 Ctrl+V를 누를 때마다 다음 항목이 붙여넣어진다.

### 1.2 핵심 가치
| 가치 | 설명 |
|------|------|
| **순서 보장** | A→B→C 복사 시 붙여넣기도 A→B→C 순서 (FIFO) |
| **제로 설정** | 별도 토글/모드 전환 없이 항상 자동 동작 |
| **직관적 UI** | 복사 시 패널이 자동으로 나타나 현재 상태를 즉시 확인 가능 |
| **노션 호환** | 마크다운 서식 + 이미지 혼합 복사 완벽 지원 |

### 1.3 범위
| 포함 | 제외 |
|------|------|
| Windows 10/11 | Mac / Linux |
| 텍스트, 이미지, Rich Text, HTML | 파일/폴더 경로 복사 |
| 노션 복사 내용 서식 보존 | 노션 API 연동 |
| 로컬 SQLite 영구 저장 | 클라우드 동기화 |
| 단일 .exe 배포 | OCR (이미지 내 텍스트) |

### 1.4 이전 버전 교훈
> ⚠️ 이전 2회 시도(ClipStack, ClipFlow)에서 **순차 붙여넣기가 한 번도 정상 동작하지 않았음**.
> 증상: 복사한 A,B,C 중 마지막 C만 반복 붙여넣기 되거나, 아예 붙여넣기가 안 됨.
> 원인: Ctrl+V 키를 가로채는(hook) 방식의 타이밍 문제.
> **해결책: 클립보드 데이터 교체 방식 채택** — Ctrl+V를 가로채지 않고, 붙여넣기 직전에 클립보드 내용만 교체.

## 2. 핵심 동작 원리 — 순차 붙여넣기

### 2.1 클립보드 데이터 교체 방식 (Core Mechanism)

**원칙**: Ctrl+V 키 이벤트를 가로채지(intercept) 않는다. 대신 Ctrl+V가 눌리는 순간, OS가 붙여넣기를 실행하기 **직전에** 시스템 클립보드의 내용을 다음 항목으로 교체한다.

```
┌─────────────┐     ┌──────────────────┐     ┌─────────────────┐
│ 사용자가     │     │ PasteFlow가       │     │ OS가 기본        │
│ Ctrl+V 누름  │ ──→ │ 클립보드 내용을    │ ──→ │ Ctrl+V 붙여넣기  │
│              │     │ 큐의 다음 항목으로  │     │ 실행             │
│              │     │ 교체              │     │                  │
└─────────────┘     └──────────────────┘     └─────────────────┘
```

### 2.2 상태 흐름도

```
[대기 상태] ──(사용자 복사)──→ [큐 누적]
     ↑                            │
     │                       (Ctrl+V)
     │                            ↓
     │                    [클립보드 교체 → 붙여넣기 → 포인터 전진]
     │                            │
     │                   ┌────────┴────────┐
     │                   │                 │
     │            (큐 남음)          (큐 소진)
     │                   │                 │
     │                   ↓                 ↓
     │            [다음 대기]      [개입 중단 → OS 기본 동작]
     │                                     │
     └──────────(새 복사 발생)──────────────┘
```

### 2.3 구체적 동작 시퀀스

```
1. 사용자가 "안녕" 복사 → 큐: [안녕],         포인터: 0
2. 사용자가 "하세요" 복사 → 큐: [안녕, 하세요],  포인터: 0 (리셋)
3. 사용자가 "반갑습니다" 복사 → 큐: [안녕, 하세요, 반갑습니다], 포인터: 0 (리셋)

4. Ctrl+V (1회차):
   a. keyboard 라이브러리가 Ctrl+V 키다운 감지
   b. PasteFlow가 클립보드에 큐[0]="안녕"을 즉시 설정 (win32clipboard)
   c. 키 이벤트를 차단하지 않고 그대로 통과시킴
   d. OS의 기본 Ctrl+V가 클립보드의 "안녕"을 대상 앱에 붙여넣기
   e. 포인터 → 1

5. Ctrl+V (2회차):
   → 클립보드에 "하세요" 설정 → OS가 "하세요" 붙여넣기 → 포인터 → 2

6. Ctrl+V (3회차):
   → 클립보드에 "반갑습니다" 설정 → OS가 "반갑습니다" 붙여넣기 → 포인터 → 3

7. Ctrl+V (4회차 이후):
   → 포인터(3) ≥ 큐 길이(3) → PasteFlow 개입 안 함
   → 클립보드에는 마지막 "반갑습니다"가 그대로 남아있음
   → OS 기본 동작으로 "반갑습니다" 반복 붙여넣기

8. 사용자가 새로운 항목 "감사합니다" 복사:
   → 붙여넣기 진행 중(pointer>0)이므로 큐 리셋
   → 큐: [감사합니다], 포인터: 0
   → 새로 복사한 항목부터 순차 붙여넣기 시작
   → (기존 "안녕", "하세요", "반갑습니다"는 히스토리(DB)에 유지)

9. 이어서 "고맙습니다" 복사 (붙여넣기 전 연속 복사):
   → 아직 붙여넣기 안 했으므로(pointer==0) 누적
   → 큐: [감사합니다, 고맙습니다], 포인터: 0
```

### 2.4 핵심 구현 주의사항

| # | 주의사항 | 설명 |
|---|---------|------|
| 1 | **Ctrl+V를 절대 차단(block/suppress)하지 않는다** | 키 이벤트를 가로채면 타이밍 문제로 붙여넣기 실패. 반드시 이벤트를 통과시킨다 |
| 2 | **클립보드 교체는 키다운 시점에 즉시 실행** | 키업이 아닌 키다운(key_down) 시점에 클립보드 내용을 교체해야 OS가 올바른 내용을 붙여넣음 |
| 3 | **클립보드 교체 시 자체 트리거 방지** | PasteFlow가 클립보드에 쓸 때 자신의 모니터가 이를 새 복사로 인식하지 않도록 내부 플래그 처리 |
| 4 | **모든 클립보드 형식 보존** | 텍스트뿐 아니라 HTML, RTF, 이미지 등 원본 형식 그대로 클립보드에 복원 |
| 5 | **큐 리셋은 붙여넣기 진행 중 새 복사 시** | 붙여넣기 전 연속 복사는 누적, 붙여넣기 진행 중(pointer>0) 새 복사 시 큐 리셋 후 새 항목부터 시작 |

## 3. 사용 시나리오

### 시나리오 A — 기본 순차 붙여넣기
1. 엑셀에서 이름 "김철수"를 Ctrl+C → 클립보드 창에 기록
2. 이메일 "kim@test.com"을 Ctrl+C → 클립보드 창에 기록
3. 전화번호 "010-1234-5678"을 Ctrl+C → 클립보드 창에 기록
4. 입력 폼으로 이동 후 이름 필드에서 Ctrl+V → "김철수" 붙여넣기
5. 이메일 필드에서 Ctrl+V → "kim@test.com" 붙여넣기
6. 전화번호 필드에서 Ctrl+V → "010-1234-5678" 붙여넣기

### 시나리오 B — 노션 복사
1. 노션에서 마크다운 서식이 포함된 텍스트 블록 + 이미지를 Ctrl+C
2. PasteFlow가 HTML + 이미지 데이터를 함께 캡처하여 클립보드 창에 기록
3. 다른 노션 페이지에서 Ctrl+V → 서식과 이미지 그대로 붙여넣기

### 시나리오 C — 다중 선택 복사
1. 클립보드 패널에서 3개 항목을 Ctrl+클릭으로 선택
2. Ctrl+C → 선택 항목이 줄바꿈으로 결합된 새 항목 생성
3. 새 항목이 클립보드 창에 기록되고 큐 포인터 리셋
4. Ctrl+V로 결합된 텍스트 붙여넣기

### 시나리오 D — 고정 항목 사용
1. 자주 쓰는 이메일 서명을 클립보드 패널에서 우클릭 → 고정
2. 고정 항목은 패널 상단에 항상 표시
3. 고정 항목 클릭 시 해당 내용을 클립보드에 복사 → 큐에 추가

### 시나리오 E — 이미지 복사/붙여넣기
1. 웹 브라우저에서 이미지를 우클릭 → 이미지 복사
2. 클립보드 창에 썸네일로 표시
3. Ctrl+V로 이미지를 다른 앱에 붙여넣기 (원본 품질 유지)

## 4. 기능 요구사항

### F1. 클립보드 모니터링

| ID | 요구사항 |
|----|---------|
| F1-1 | 백그라운드에서 클립보드 변경 감지 (`WM_CLIPBOARDUPDATE` 이벤트 기반) |
| F1-2 | 지원 형식: 유니코드 텍스트(`CF_UNICODETEXT`), 이미지(`CF_DIB`/`CF_BITMAP`), Rich Text(`CF_RTF`), HTML(`CF_HTML`) |
| F1-3 | 노션 복사 시 HTML + 이미지 데이터를 함께 캡처 |
| F1-4 | 동일 내용 연속 복사 시 중복 항목 추가 안 함 |
| F1-5 | PasteFlow 자체의 클립보드 쓰기는 히스토리에 추가 안 함 (내부 플래그 `_self_triggered`) |
| F1-6 | 히스토리 최대 50개 유지 (FIFO), 초과 시 가장 오래된 비고정 항목 삭제 |
| F1-7 | Ctrl+C뿐 아니라 모든 클립보드 변경 감지 (우클릭 복사, 앱 내부 복사 등) |

### F2. 순차 붙여넣기 (핵심 기능)

| ID | 요구사항 |
|----|---------|
| F2-1 | **항상 활성** — 별도 모드 토글 없음. 프로그램 실행 중 항상 순차 붙여넣기 동작 |
| F2-2 | **FIFO 순서** — 먼저 복사한 항목이 먼저 붙여넣어짐 |
| F2-3 | **클립보드 교체 방식** — Ctrl+V 키다운 감지 시 클립보드 내용을 큐의 현재 포인터 항목으로 교체한 뒤, 키 이벤트를 그대로 통과시킴 |
| F2-4 | **큐 포인터** — 붙여넣기할 때마다 포인터가 1 전진 |
| F2-5 | **큐 소진 후** — 포인터가 큐 길이 이상이면 개입하지 않음. 마지막 항목이 클립보드에 남아 OS 기본 동작으로 반복 붙여넣기 |
| F2-6 | **큐 리셋** — 붙여넣기 진행 중(pointer>0) 새 복사 시 큐를 비우고 새 항목부터 시작. 붙여넣기 전(pointer==0) 연속 복사는 큐에 누적. 히스토리(DB)에는 모든 항목 유지 |
| F2-7 | **패널에 진행 상태 표시** — 히스토리 섹션 헤더에 현재 포인터 위치와 전체 큐 크기 (예: "붙여넣기 1/3") |
| F2-8 | **모든 클립보드 형식 보존** — 텍스트, HTML, RTF, 이미지 등 원본 형식 그대로 클립보드에 복원하여 붙여넣기 |

### F3. 패널 자동 표시 (복사 알림)

| ID | 요구사항 |
|----|---------|
| F3-1 | 클립보드 변경(복사) 발생 시 패널이 자동으로 표시됨 |
| F3-2 | 패널 자동 표시 시 포커스를 빼앗지 않음 (`SW_SHOWNOACTIVATE`) — 현재 작업 방해 없음 |
| F3-3 | 사용자가 Alt+V로 직접 패널을 열 경우에는 포커스 이동 (활성 창으로 동작) |

### F4. 전체 클립보드 패널

| ID | 요구사항 |
|----|---------|
| F4-1 | 단축키(기본: `Alt+V`)로 팝업/닫기 토글 |
| F4-2 | 시스템 트레이 아이콘 클릭으로 팝업 |
| F4-3 | 복사 발생 시 패널이 자동으로 팝업 (F3-1 참고) |
| F4-4 | 패널 구성: 상단 고정 섹션 + 하단 히스토리 섹션 |
| F4-5 | 각 항목 표시: 번호, 유형 아이콘, 미리보기(텍스트 2줄 or 썸네일), 복사 시간 |
| F4-6 | `Ctrl+클릭`: 다중 선택 (비연속) |
| F4-7 | `Shift+클릭`: 범위 선택 (연속) |
| F4-8 | 항목 우클릭 컨텍스트 메뉴: 붙여넣기 / 고정·해제 / 복사 / 삭제 |
| F4-9 | 패널 외부 클릭 시 자동 닫기 (설정으로 비활성화 가능) |
| F4-10 | 상단 검색 박스로 히스토리 텍스트 검색 |
| F4-11 | 텍스트 항목: 최대 2줄 미리보기, 너무 긴 경우 말줄임(`...`) 처리 |
| F4-12 | 이미지 항목: 썸네일(80×60px)로 직관적 표시, 마우스 오버 시 확대 미리보기 |

### F5. 항목별 개별 붙여넣기

| ID | 요구사항 |
|----|---------|
| F5-1 | `Alt+1`~`Alt+9`: 히스토리 1~9번 항목을 클립보드에 설정 후 즉시 붙여넣기 (패널 닫힌 상태에서도 동작) |
| F5-2 | 패널에서 항목 더블클릭: 해당 항목을 클립보드에 설정 후 이전 포커스 앱에 붙여넣기, **패널 유지** |
| F5-3 | 개별 붙여넣기는 순차 큐 포인터에 영향을 주지 않음 |
| F5-4 | 붙여넣기는 대상 앱에 포커스를 넘긴 후 실행 (`SendInput` API 사용) |

### F6. 다중 선택 및 복사

| ID | 요구사항 |
|----|---------|
| F6-1 | `Ctrl+클릭`/`Shift+클릭`으로 복수 항목 선택 |
| F6-2 | 선택된 항목들을 `Ctrl+C` → 줄바꿈(`\n`)으로 결합하여 새 항목 생성 |
| F6-3 | 결합된 텍스트를 클립보드에 쓰고 히스토리에 신규 항목 등록 |
| F6-4 | 이미지 포함 시: 텍스트 항목은 결합, 이미지는 `[이미지]` 플레이스홀더. 전체가 이미지면 첫 이미지를 클립보드에 복사 |
| F6-5 | 새 항목 생성 시 클립보드 모니터의 자체 트리거 방지 플래그 사용 |

### F7. 고정(Pin) 기능

| ID | 요구사항 |
|----|---------|
| F7-1 | 항목 우클릭 → "고정" 또는 항목 내 핀(📌) 버튼 클릭 |
| F7-2 | 고정 항목은 패널 상단 고정 섹션에 표시 |
| F7-3 | 고정 항목은 50개 히스토리 한도에서 제외 |
| F7-4 | 재시작 후에도 고정 항목 유지 (SQLite 영구 저장) |
| F7-5 | 고정 항목 순서를 드래그로 재정렬 가능 |
| F7-6 | 고정 항목 클릭 시 해당 내용을 클립보드에 복사 (큐에 추가됨) |

### F8. 이미지 지원

| ID | 요구사항 |

### 6.4 순차 붙여넣기 핵심 로직 (의사코드)

```python
class PasteQueue:
    """순차 붙여넣기 큐 관리"""
    def __init__(self):
        self.queue = []      # ClipboardItem 리스트
        self.pointer = 0     # 다음 붙여넣기할 인덱스

    def add_item(self, item):
        """새 항목 추가 — 진행 중이면 큐 리셋, 아니면 누적"""
        if self.pointer > 0:
            self.queue.clear()  # 붙여넣기 진행 중 → 큐 리셋
        self.queue.append(item)
        self.pointer = 0

    def get_next(self):
        """다음 붙여넣기 항목 반환. 소진 시 None"""
        if self.pointer < len(self.queue):
            item = self.queue[self.pointer]
            self.pointer += 1
            return item
        return None  # 큐 소진 → 개입 안 함


class PasteInterceptor:
    """Ctrl+V 감지 → 클립보드 교체"""
    def __init__(self, paste_queue):
        self.queue = paste_queue
        self._self_triggered = False

    def on_ctrl_v_keydown(self):
        """Ctrl+V 키다운 시 호출 (키를 차단하지 않음!)"""
        next_item = self.queue.get_next()
        if next_item is None:
            return  # 큐 소진 → 아무것도 안 함 → OS 기본 동작

        # 자체 트리거 방지 플래그 설정
        self._self_triggered = True

        # 클립보드에 다음 항목 설정 (모든 형식 보존)
        set_clipboard(next_item)  # win32clipboard 사용

        self._self_triggered = False
        # 키 이벤트는 차단하지 않고 그대로 통과 → OS가 붙여넣기 실행
```

## 10. 이전 버전 교훈 & 주의사항

| # | 교훈 | PasteFlow 대응 |
|---|------|----------------|
| 1 | Ctrl+V를 가로채는(hook/suppress) 방식은 타이밍 문제로 실패 | 클립보드 데이터 교체 방식 채택. 키 이벤트는 절대 차단하지 않음 |
| 2 | 순차 모드 토글이 혼란을 줌 | 항상 활성 방식으로 변경. 모드 개념 제거 |
| 3 | 디자인이 직관적이지 않았음 | Catppuccin Mocha 테마, 상태별 색상 구분, 좌측 강조 바로 현재 상태 즉시 파악 |
| 4 | 기능이 많아 MVP 검증이 늦어짐 | Phase 1에서 순차 붙여넣기만 집중 검증 후 확장 |
| 5 | 자체 클립보드 쓰기가 모니터에 재감지됨 | `_self_triggered` 플래그로 자체 트리거 완전 차단 |

# 드래그 붙여넣기 후 패널 닫힘 수정계획

## 문제 현상
- 패널에서 텍스트 항목을 드래그해서 외부 앱에 붙여넣기하면 패널이 사라짐
- 재현 조건: 항상 발생
- 사용자가 목격한 증상: 드래그-붙여넣기 한 번이면 패널이 닫혀 연속 붙여넣기 불가

## 원인 분석

### 1순위: `_on_drag_to_app`에서 명시적 `panel.hide()` 호출
- 파일: `pasteflow/main.py` L289
- 설명: 드래그-붙여넣기 핸들러가 `WM_PASTE` 전송 직전에 `self.panel.hide()`를 직접 호출하고 있음. 이게 진짜 원인. 이전 시도에서 `changeEvent` 쪽만 막았지만 이 명시적 hide를 놓쳤음.

```python
def _on_drag_to_app(self, item_id: int, cursor_pos):
    ...
    self.interceptor._set_clipboard(full_item)
    self.panel.hide()   # ← 여기서 직접 hide
    ...
    win32gui.SendMessage(target, win32con.WM_PASTE, 0, 0)
```

### 2순위: `mouseReleaseEvent`에서 `panel._ext_drag_active` 리셋 타이밍 문제
- 파일: `pasteflow/ui/panel.py` L301~307 (mouseReleaseEvent)
- 설명: 드래그 종료 시 `panel._ext_drag_active = False`를 signal 발생 **전에** 리셋하고 있음. 이후 외부 앱이 활성화되면서 `ActivationChange` 이벤트가 오면 `changeEvent`의 guard가 이미 해제되어 패널이 닫힐 수 있음.

```python
def mouseReleaseEvent(self, event):
    if self._ext_drag_active:
        self._ext_drag_active = False
        panel._ext_drag_active = False   # ← guard 해제
        ...
        self.external_drag_paste.emit(...)  # ← 이 시그널 처리 후 OS 이벤트가 들어와 changeEvent 발동
```

## 단계별 수정 계획

> 각 Phase는 새로운 대화세션에서 시작한다.
> 중단되어도 다음 세션에서 이 파일을 열어 이어서 진행 가능.

### Phase 1 — 명시적 hide 제거 (핵심)
- [ ] `main.py` L289: `self.panel.hide()` 한 줄 제거

- [ ] 커밋: `Fix: 드래그 붙여넣기 후 패널 유지 - 명시적 hide 제거`

### Phase 2 — changeEvent guard 타이밍 보완
- [ ] `panel.py` `mouseReleaseEvent`: `panel._ext_drag_active = False` 즉시 리셋 대신 `QTimer.singleShot(300, lambda: setattr(panel, '_ext_drag_active', False))` 로 변경

- [ ] 수동 확인: 300ms 후 패널이 자동닫기 설정에 따라 정상 동작하는지 확인
